# HaTickets v0.3.0（用户版发布说明）

发布日期：2026-03-31

这版的目标很直接：让普通用户更容易跑通 Mobile 抢票流程，并减少误操作。

## 你会直接感受到的变化

- 现在主推 **安卓 Mobile 方案**，文档和流程都围绕真机抢票优化。
- 新增自然语言入口，可以先用一句话生成配置并做安全探测。
- 抢票日志更清晰，成功/失败提示更直观。
- 对“已有未支付订单”的场景做了更友好的提示，不再容易误判。

## 推荐使用流程（3 步）

1. 启动 Appium  
   `./mobile/scripts/start_appium.sh`
2. 先做安全探测（不提交订单）  
   `./mobile/scripts/start_ticket_grabbing.sh --probe --yes`
3. 正式抢票  
   `./mobile/scripts/start_ticket_grabbing.sh --yes`

## 自然语言入口（可选）

你也可以先让脚本自动识别配置：

```bash
./mobile/scripts/run_from_prompt.sh --mode probe --yes "给张三和李四抢4 月 6 号张杰的北京站演唱会内场门票，票价 1680 元"
```

说明：
- `summary`：只看匹配结果
- `apply`：写配置，不执行探测
- `probe`：写配置并执行安全探测（推荐）

## 需要特别注意的变化

- `Desktop` 方案当前不再作为可执行主方案，请优先使用 Mobile。
- `mobile/config.jsonc` 现在是本地配置文件，不再提交到仓库。
- `run_from_prompt` 不会直接帮你正式提交订单；正式抢票请显式执行 `start_ticket_grabbing.sh --yes`。

## 已知边界

- 线上库存、风控、验证码会影响结果，自动化不能保证 100% 成功。
- 建议每次正式抢票前，先跑一次 `--probe` 确认页面和配置都正常。
