# HaTickets v0.3.0 Release Notes

发布日期：2026-03-31  
对比基线：`v0.2.0 (92ad0ab)` → `master (3129d3c)`  
变更规模：48 commits，45 files changed（`+12921 / -517`）

## 版本定位

`v0.3.0` 的核心目标是把项目从“多方案并行”收敛为“Mobile 真机优先”，并显著提升实战稳定性、可观测性和上手效率。

## 主要功能提升

### 1) Mobile 主流程升级（实战稳定性）

- 强化 `DamaiBot` 抢票主流程，完善页面状态识别、重试恢复与结果判定。
- 新增“未支付订单”识别逻辑：检测到已有未支付订单时，会按成功路径提示用户立即支付，避免误判为失败。
- 增强开售前后等待与快速重试策略，降低开抢瞬间状态抖动导致的失败概率。

涉及文件：
- `mobile/damai_app.py`
- `tests/unit/test_mobile_damai_app.py`
- `tests/integration/test_mobile_workflow.py`

### 2) 自然语言入口（Prompt 驱动）

- 新增自然语言入口：
  - `./mobile/scripts/run_from_prompt.sh --mode summary|apply|probe --yes "..."`
- 新增提示词解析器，支持从中文表达提取：
  - 观演人、日期、票价偏好、关键词/演出信息。
- 新增强校验与防误操作机制：
  - 缺少关键信息时直接终止并给出规范提示词模板。
  - 人数和购票数量冲突时直接阻断，避免误下单。
- `config.jsonc` 缺失时，`run_from_prompt` 会自动从模板创建再继续执行。

涉及文件：
- `mobile/prompt_parser.py`
- `mobile/prompt_runner.py`
- `mobile/scripts/run_from_prompt.sh`
- `tests/unit/test_mobile_prompt_parser.py`
- `tests/unit/test_mobile_prompt_runner.py`

### 3) item_url 解析与自动导航

- 新增 `item_url/item_id` 解析能力，可自动提取演出信息并补足搜索关键词。
- 增强首页/搜索页到目标演出的自动导航链路，减少手动操作依赖。

涉及文件：
- `mobile/item_resolver.py`
- `mobile/damai_app.py`
- `tests/unit/test_mobile_item_resolver.py`

### 4) 性能与压测能力

- 新增热路径压测：
  - `./mobile/scripts/benchmark_hot_path.sh`
- 支持步骤级耗时输出（step timeline），用于定位瓶颈。
- `rush_mode` 优化实战路径，减少非必要 UI 读取与等待。
- 压测起跑规则明确为从 `detail_page` 开始，结果更可比。

涉及文件：
- `mobile/hot_path_benchmark.py`
- `mobile/scripts/benchmark_hot_path.sh`
- `tests/unit/test_mobile_hot_path_benchmark.py`

### 5) 运行模式与脚本体验改进

- `start_ticket_grabbing.sh` 增加明确模式语义：
  - `--probe`：安全探测（不提交订单）
  - 默认：正式抢票（会尝试提交）
- 当配置中的 `probe_only / if_commit_order` 与命令意图冲突时，脚本会醒目提示，并支持自动改写配置后继续。

涉及文件：
- `mobile/scripts/start_ticket_grabbing.sh`
- `mobile/config.py`
- `tests/unit/test_mobile_config.py`

### 6) 文档与合规补强

- 文档结构重写为 Mobile 优先，显式强调 Desktop 方案当前不可用。
- 增加中英双语免责声明、LICENSE、NOTICE，明确“仅供学习研究使用”边界。

涉及文件：
- `README.md`
- `docs/quick-start.md`
- `docs/mobile-ticket-logic.md`
- `docs/project-overview.md`
- `DISCLAIMER.md`
- `LICENSE`
- `NOTICE`

## 兼容性与行为变化（升级必读）

### 1) Desktop 路线降级为历史方案

- Desktop 文档仍保留，但不再作为当前可执行主方案。

### 2) `mobile/config.jsonc` 不再纳入版本跟踪

- `mobile/config.jsonc` 已从仓库跟踪中移除并加入 `.gitignore`。
- 本地配置建议：
  - 普通用户：维护 `mobile/config.jsonc`
  - 开发者：显式使用 `mobile/config.local.jsonc` + `--config` 或 `HATICKETS_CONFIG_PATH`

### 3) Prompt 流程不会直接执行正式提交

- `run_from_prompt` 的 `probe` 模式用于“写配置 + 安全探测”。
- 正式抢票需显式执行：
  - `./mobile/scripts/start_ticket_grabbing.sh --yes`

## 从 v0.2.0 升级到 v0.3.0（建议步骤）

1. 更新代码并安装依赖  
   `poetry install`
2. 准备配置（如果本地不存在）  
   `cp mobile/config.example.jsonc mobile/config.jsonc`
3. 启动 Appium  
   `./mobile/scripts/start_appium.sh`
4. 先做安全探测（推荐）  
   `./mobile/scripts/start_ticket_grabbing.sh --probe --yes`
5. 再执行正式抢票  
   `./mobile/scripts/start_ticket_grabbing.sh --yes`

## 已知边界

- 线上风控与库存波动仍会影响最终结果，自动化只能提升效率，不能保证 100% 成功。
- 页面结构更新可能导致定位失效，建议优先用 `--probe` 验证后再正式执行。
